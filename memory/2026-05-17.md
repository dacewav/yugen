# 2026-05-17 — Scaffolding inicial + Fase 1

## Qué se hizo (scaffolding)

- Proyecto SvelteKit 2 + Svelte 5 + TypeScript desde cero
- Server modules: firebase-admin, r2, stripe, mercadopago, resend, auth, rate-limit, file-validation
- API endpoints: auth (login/logout/session), stripe webhook, mercadopago webhook, r2 (upload-url/confirm), orders
- Tipos: member, content (beat/drumkit/service), order
- Stores: auth, cart
- Validators: Zod schemas para orders, uploads, contact, newsletter, beats
- Formatters: price, date, duration, number, slug, order status
- hooks.server.ts: Firebase Auth con session cookies + claims version check
- Páginas: home, login, beats (list + detail), drumkits, services, drops, contact, admin dashboard, portal del cliente
- Firestore rules + indexes
- Design system CSS (dark theme: bg #030305, cyan, sakura, neon)
- Cloudflare adapter parcheado para externalizar Node built-ins
- Build exitoso, 0 errores TS

## Qué se hizo (Fase 1 — datos)

- **Client type** (`src/lib/types/client.ts`): uid, email, displayName, photoURL, purchaseHistory, favorites, createdAt, lastLoginAt
- **Client auto-creation**: login endpoint crea/actualiza documento `clients/{uid}` en Firestore al autenticar
- **hooks.server.ts**: detecta clientes vs crew via `isClient` flag (sin custom claims = cliente)
- **app.d.ts**: `isClient: boolean` en `App.Locals.user`
- **queries.ts** (`src/lib/server/queries.ts`):
  - `getBeats(filters)` — género, mood, BPM range, precio range, key, sort (recent/popular/price_asc/price_desc), search por nombre/tags/género
  - `getBeatBySlug(slug)`, `getFeaturedBeats(limit)`
  - `getDrumkits()`, `getDrumkitBySlug(slug)`
  - `getServices()`, `getServiceBySlug(slug)`
  - `getDrops()`
  - `getClient(uid)`, `getClientOrders(uid)`
  - `reserveBeat(beatId, uid)` — transacción atómica, lock 5min
  - `releaseBeat(beatId, uid)` — libera reserva
  - `markBeatSold(beatId, buyerUid, orderId)` — transacción atómica
  - `cleanupExpiredReservations()` — batch cleanup
- **Exclusividad en Beat**: campos `isExclusive`, `isSold`, `soldTo`, `soldAt`, `soldOrderId`, `reservedBy`, `reservedUntil`
- **Server loads reales**: beats, drumkits, services, drops — todos con queries a Firestore + fallback a `[]`
- **Beat detail**: badge exclusivo, overlay "VENDIDO", mood tags
- **Beats page**: search bar, filtros por género/mood con pills, sort options, todo como query params
- **API endpoints**: `POST /api/beats/reserve`, `POST /api/beats/release`
- **Páginas cliente**: `/account` (perfil), `/account/orders` (historial)
- **Seed script** (`scripts/seed.mjs`): 10 beats, 3 drumkits, 3 servicios, 1 cliente de prueba
- **Firestore rules**: colección `clients/{uid}` con permisos (owner + editor)
- **Firestore indexes**: composite indexes para beats (plays, price, featured, slug), orders (clientId), drops, drumkits, services
- 0 errores TS, build Cloudflare OK

## Problemas resueltos (scaffolding)

- SvelteKit 2.60.1 bug con guard plugin → downgrade a 2.21.0
- Firebase Admin eager init → lazy init con `process.env`
- Cloudflare adapter no externaliza `http`, `https`, `net`, `tls` → parche postinstall
- Resend SDK `reply_to` vs `replyTo` → fix

## Problemas resueltos (Fase 1)

- `svelte-check` error por tipos `Record<string, unknown>` → cast `as any` en server loads (patrón consistente con portal page)
