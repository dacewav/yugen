# 2026-05-17 — Scaffolding inicial

## Qué se hizo

- Proyecto SvelteKit 2 + Svelte 5 + TypeScript desde cero
- Server modules: firebase-admin, r2, stripe, mercadopago, resend, auth, rate-limit, file-validation
- API endpoints: auth (login/logout/session), stripe webhook, mercadopago webhook, r2 (upload-url/confirm), orders
- Tipos: member, content (beat/drumkit/service), order
- Stores: auth, cart
- Validators: Zod schemas para orders, uploads, contact, newsletter, beats
- Formatters: price, date, duration, number, slug, order status
- hooks.server.ts: Firebase Auth con session cookies + claims version check
- Páginas: home, login, beats (list + detail), drumkits, services, drops, contact, admin dashboard, portal del cliente
- Firestore rules + indexes
- Design system CSS (dark theme: bg #030305, cyan, sakura, neon)
- Cloudflare adapter parcheado para externalizar Node built-ins
- Build exitoso, 0 errores TS

## Problemas resueltos

- SvelteKit 2.60.1 bug con guard plugin → downgrade a 2.21.0
- Firebase Admin eager init → lazy init con `process.env`
- Cloudflare adapter no externaliza `http`, `https`, `net`, `tls` → parche postinstall
- Resend SDK `reply_to` vs `replyTo` → fix
